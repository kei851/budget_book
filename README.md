# 💰 Budget Book - 楽天カード家計簿アプリ

楽天カードの利用明細CSVファイルを読み込んで、月毎の支出を可視化・分析する家計簿Webアプリケーション

![App Screenshot](docs/screenshot.png)

## 🎯 プロジェクト概要

楽天カード利用者が自身の支出データを効率的に可視化・分析できるWebアプリケーション。
Vue.js + Rails API + SQLiteでフルスタック実装完了。

## ✨ 完成機能

### 📊 データ可視化
- **月次支出グラフ**: Chart.jsによる積み上げ棒グラフ
- **カテゴリ別円グラフ**: 支出割合の可視化
- **日別推移線グラフ**: 日次支出の推移表示
- **統計サマリー**: 総支出、平均支出、最大支出、取引件数

### 🗂️ データ管理
- **CSVインポート**: BOM付きUTF-8、半角カタカナ対応
- **カテゴリ自動分類**: 店舗名による自動カテゴリ振り分け
- **月選択ナビゲーション**: 矢印での月移動＋クリック選択
- **データがある月のハイライト**: 視認性の向上

### 🎨 ユーザーインターフェース
- **レスポンシブデザイン**: モバイル・タブレット・デスクトップ対応
- **直感的な操作**: ドラッグ&ドロップ、ワンクリック月選択
- **リアルタイム更新**: データ変更の即座反映
- **プライバシー保護**: 金額非表示モード搭載
- **横スクロールグラフ**: 12ヶ月以上のデータ対応
- **カテゴリ手動変更**: ドロップダウンでリアルタイム分類変更
- **高速ナビゲーション**: キャッシュ機能によるスムーズな月移動

## 🚀 セットアップ・使用方法

### 1. プロジェクトクローン
```bash
git clone <repository-url>
cd budget_book
```

### 2. バックエンド起動
```bash
cd backend
bundle install
rails db:migrate
rails db:seed
rails server
```

### 3. フロントエンド起動
```bash
cd frontend
npm install
npm run dev
```

### 4. アプリケーション利用
1. ブラウザで http://localhost:3002 にアクセス
2. 楽天カードCSVファイルをアップロード
3. データが自動で可視化される
4. 「詳細分析を見る」で詳細画面へ
5. 月選択で過去データを確認

## 📄 対応CSVフォーマット

### 楽天e-NAVIの利用明細CSV
```csv
"利用日","利用店名・商品名","利用者","支払方法","利用金額","支払手数料","支払総額","8月支払金額","9月繰越残高","新規サイン"
"2024/07/31","ﾄｳﾀﾞｲﾁﾕｳｵｳｼﾖｸﾄﾞｳ","本人","1回払い","594","0","594","594","0","*"
```

### 対応機能
- **エンコーディング**: BOM付きUTF-8、Shift_JIS自動変換
- **文字種**: 半角カタカナ→全角カタカナ変換
- **エラーハンドリング**: 不正データのスキップ
- **大容量対応**: 数千件のデータ処理

## 🏷️ カテゴリ自動分類

### 分類ルール
```ruby
{
  '食費' => ['セブン-イレブン', 'マクドナルド', 'マツヤ', 'ホットモット'],
  '交通費' => ['トウダイノウガク', 'メトロ', 'トウダイ'],
  '日用品費' => ['ダイソー', 'ドラッグストア'],
  '娯楽費' => ['映画', 'カラオケ', 'ゲーム'],
  # ...その他カテゴリ
}
```

### 分類方法
1. **店舗名マッチング**: 部分一致による自動分類
2. **半角カタカナ対応**: NKFによる文字変換
3. **優先度制御**: 複数該当時の分類ルール
4. **未分類対応**: 該当なしの場合の処理

## 🐛 トラブルシューティング

### よくある問題

#### CSVアップロードエラー
```
問題: "CSVインポートに失敗しました"
解決: ファイルのエンコーディングを確認（BOM付きUTF-8推奨）
```

#### グラフが表示されない
```
問題: Chart.jsが動作しない
解決: ブラウザのJavaScriptが有効か確認
```

#### CORS エラー
```
問題: "Access-Control-Allow-Origin"エラー
解決: backend/config/initializers/cors.rb を確認
```
