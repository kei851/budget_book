# 家計簿アプリ 要件定義書

**バージョン**: 1.0  
**作成日**: 2025年8月10日  
**更新日**: 2025年8月10日  

## 1. プロジェクト概要

### 1.1 背景・目的
楽天カード利用者が自身の支出状況を効率的に把握し、家計管理を改善することを目的とする。従来のCSVファイルをExcelで開く作業を自動化し、直感的で視覚的な支出分析を提供する。

### 1.2 プロジェクト名
**家計簿アプリ - 楽天カード明細可視化システム**

### 1.3 プロジェクトの目標
楽天カードの利用明細CSVファイルを読み込み、カテゴリ別に自動分類して月毎の支出を積み上げ棒グラフで可視化する家計簿アプリケーション

## 2. ステークホルダー

### 2.1 主要ステークホルダー
- **プロダクトオーナー**: 個人開発者
- **エンドユーザー**: 楽天カード利用者（家計管理に関心のある個人）
- **開発者**: 個人開発者（学習目的を含む）

### 2.2 ユーザーペルソナ
- **主要ユーザー**: 楽天カードを利用している20-50代の個人
- **特徴**: 家計管理に関心があり、定期的に支出を確認したい
- **スキルレベル**: 基本的なPC操作ができる（CSVファイルのダウンロード等）
- **利用環境**: デスクトップブラウザ、タブレット、スマートフォン

## 3. ビジネス要件

### 3.1 主要ビジネス価値
1. **効率化**: 手動Excel作業の自動化による時間短縮
2. **視覚化**: グラフによる直感的な支出把握
3. **分析**: カテゴリ別・月毎の支出パターン分析
4. **学習**: フロントエンド・バックエンド技術の習得

### 3.2 主要ユースケース
1. **月毎支出の可視化**: 過去1年間の支出推移をグラフで把握
2. **カテゴリ別分析**: 食費、交通費等の内訳を確認
3. **詳細データ確認**: 特定の取引や期間の明細を検索・閲覧
4. **支出パターン分析**: 月平均、最大支出月等の統計情報を把握

### 3.3 成功指標
- CSVファイル処理の成功率: 95%以上
- グラフ表示までの処理時間: 30秒以内（20年分データ）
- カテゴリ自動分類の精度: 85%以上
- ユーザー満足度: 個人利用での実用性確認

## 4. 機能要件

### 4.1 必須機能（Phase 1）✅実装完了
1. **F001: CSV読み込み機能** ✅
   - 楽天カードCSVファイルのドラッグ&ドロップアップロード
   - ファイル形式・エンコーディング検証（BOM付きUTF-8、Shift_JIS対応）
   - データ解析・変換・エラーハンドリング

2. **F002: データ処理・集計機能** ✅
   - 利用日による月毎グルーピング
   - 支出金額の合計計算
   - 統計データ生成（平均、最大・最小、取引回数）
   - データベースによる永続化

3. **F003: カテゴリ自動分類機能** ✅
   - 店名による自動カテゴリ分類（135キーワード対応）

   - 優先順位制御機能
   - 7カテゴリ（食費、交通費、住宅費、娯楽費、投資、日用品費、その他）

4. **F004: グラフ表示機能** ✅
   - 月毎支出の積み上げ棒グラフ表示
   - カテゴリ別色分け
   - 期間選択機能（3ヶ月/6ヶ月/12ヶ月/全期間）
   - インタラクティブ操作（ホバー、クリック）

5. **F005: 基本詳細表示機能** ✅
   - 取引明細一覧表示
   - カテゴリ別フィルタリング
   - 基本的な統計サマリー

6. **F006: 詳細分析ページ機能** ✅
   - 月選択ナビゲーション（←→ボタン + ドロップダウン）
   - 統計カード表示（総支出、最高支出、平均支出、取引件数）
   - カテゴリ別円グラフ
   - 日別支出推移線グラフ
   - 取引明細一覧（金額順・日付順ソート）

## 5. 非機能要件

### 5.1 パフォーマンス要件
- **CSV読み込み時間**: 20年分データ（24万〜48万件）を30秒以内で処理
- **グラフ描画時間**: データ表示まで5秒以内
- **画面遷移**: 1秒以内の応答時間
- **メモリ使用量**: ブラウザメモリ1GB以内での動作

### 5.2 可用性要件
- **ブラウザ対応**: モダンブラウザ（Chrome, Firefox, Safari, Edge）
- **オフライン動作**: インターネット接続不要での基本機能動作
- **レスポンシブ対応**: デスクトップ、タブレット、スマートフォン対応

### 5.3 セキュリティ要件
- **個人情報保護**: 金融データの適切な取り扱い
- **データ保存**: ユーザー同意なしにサーバーへデータ永続化しない
- **通信暗号化**: データ送信時はHTTPS必須（将来の認証機能で）

### 5.4 ユーザビリティ要件
- **直感的UI**: ドラッグ&ドロップでのファイル選択
- **レスポンシブデザイン**: 画面サイズに応じた最適表示
- **進捗表示**: データ読み込み中の進捗表示
- **エラー対応**: わかりやすいエラーメッセージ

### 5.5 拡張性要件
- **新フォーマット対応**: 他社CSVフォーマットの追加が容易
- **カテゴリ拡張**: 新しいカテゴリの追加・カスタマイズが可能
- **機能拡張**: 新機能追加に対応できるアーキテクチャ

## 6. 制約事項

### 6.1 技術的制約
- **データ処理**: 大量データ（20年分）のため、必要に応じてバックエンド処理を採用
- **ブラウザ制限**: メモリ使用量とJavaScript処理性能の制限を考慮
- **CSV形式**: Phase 1では楽天カード形式のみ対応

### 6.2 リソース制約
- **開発リソース**: 個人開発（1人）
- **開発期間**: 学習を含めた段階的開発
- **予算**: 個人プロジェクトのため最小限

### 6.3 法的・コンプライアンス制約
- **個人情報保護法**: 金融データの適切な取り扱い
- **著作権**: オープンソースライブラリの適切な利用
- **利用規約**: 楽天カード利用規約に準拠

## 7. データ要件

### 7.1 対象データ
- **データソース**: 楽天カード利用明細CSV
- **データ量**: 20年分の取引明細（約24万〜48万件を想定）
- **ファイルサイズ**: 50MB〜100MBのCSVファイル処理
- **エンコーディング**: UTF-8 with BOM、Shift_JIS等に対応

### 7.2 データ構造
```
"利用日","利用店名・商品名","利用者","支払方法","利用金額","支払手数料","支払総額","支払月","8月支払金額","9月繰越残高","9月以降支払金額"
```

### 7.3 重要なデータ項目
- **利用日**: 取引日（YYYY/MM/DD形式）
- **利用店名・商品名**: 店舗名や商品情報
- **利用金額**: 支払金額（数値）
- **支払月**: 支払月情報

## 9. リスク分析

### 9.1 技術的リスク
- **リスク**: 大量データ処理によるブラウザ性能問題
- **対策**: バックエンド処理の導入、データ分割処理

### 9.2 ユーザビリティリスク
- **リスク**: CSVフォーマットの変更による読み込みエラー
- **対策**: 柔軟なCSV解析ロジック、エラーハンドリング強化

### 9.3 プロジェクトリスク
- **リスク**: 個人開発による開発速度の制限
- **対策**: 段階的開発、MVP（最小実用機能）アプローチ
