# 家計簿アプリ マスタ仕様書

> このドキュメントがアプリの唯一の正とする。機能追加・変更のたびに更新する。

---

## 1. アプリ概要

楽天カード・エポスカード・楽天銀行のCSVを取り込み、支出をカテゴリ別に可視化する個人家計簿アプリ。
AIチャットで現金支出の記録・家計アドバイスが可能。資産残高も月次で管理できる。

**スタック**
- バックエンド: Ruby on Rails 8（API mode）, SQLite, port 3001
- フロントエンド: Vue 3 Composition API + Pug + SCSS, Vite, port 3002

## 3. 各ページ機能詳細

### 3-1. ホームページ（HomePage.vue）
- 月次棒グラフ（12ヶ月分、カテゴリ積み上げ）
- サマリーカード（今月支出・前月比）
- CSVアップロードボタン → UploadManager を呼び出す

### 3-2. 詳細分析（AnalyticsPage.vue）

3タブ構成：

| タブ | 内容 |
|------|------|
| 概要 | 統計カード4枚 + 円グラフ（カテゴリ別） + 折れ線グラフ（日別） + AIサマリ |
| インサイト | 予算カード + 前月比較カード + 定期支出カード |
| 取引明細 | カテゴリ/並び順フィルタ + 取引テーブル（インラインカテゴリ変更可） |

月ナビゲーション（← 年月 →）でデータを切り替え。月ピッカーあり。

### 3-3. 資産管理（AssetPage.vue）

口座マスタ（固定6口座）:

| 口座名 | 種別 |
|--------|------|
| 楽天銀行 | 銀行 |
| ゆうちょ銀行 | 銀行 |
| 楽天証券 | 投資 |
| 現金 | 現金 |
| PayPay | 電子マネー |
| Pasmo | 電子マネー |

**月次フロー**: 月末に各口座の残高を6項目入力 → 保存 → 推移グラフに反映

表示:
- 総資産（合計） + 前月比
- 種別グループ別の内訳
- 総資産推移折れ線グラフ
- 「今月の残高を入力」ボタン（未入力月）/ 「編集」ボタン（入力済み月）

### 3-4. キーワード管理（CategoryRulesPage.vue）
- キーワード→カテゴリのルール一覧・追加・編集・削除
- 変更時にCSVファイルへ自動反映（after_save フック）

### 3-5. AIチャット（AiChatPanel.vue）
- 右下の💬ボタンでパネル開閉（デフォルト: 閉じた状態）
- **通常の家計相談**: ストリーミングで回答
- **支出の記録**: 「昨日コンビニで500円使った」と話しかけると自動でDBに保存
  → `✅ コンビニ ¥500を記録しました`（緑色バブル）

## 4. データソースと取込みルール

### 4-1. 基本原則

**「一つの支出は一つのソースで1回だけ記録する」**

カード払いは銀行CSVにも引き落とし行として出現するため、重複を防ぐフィルタが必要。

### 4-2. お金の流れ

```
┌─────────────────────────────────────────┐
│ 支出の種類           記録ソース           │
├─────────────────────────────────────────┤
│ 楽天カード払い    → 楽天カードCSV        │
│ エポスカード払い  → エポスカードCSV       │
│ JCBデビット払い   → 楽天銀行CSV（出金行） │
│ 銀行振込（家賃等）→ 楽天銀行CSV（振込行） │
│ ゆうちょ振込      → 未対応（手入力推奨）  │
│ 現金払い          → AIチャットで記録      │
│ Pasmo利用         → AIチャットで記録      │
│ PayPay利用        → AIチャットで記録      │
│ 楽天証券積立      → 楽天カードCSV（月5万）│
└─────────────────────────────────────────┘
```

### 4-3. 楽天銀行CSV フィルタリングルール

`csv_import_service.rb` の `rakuten_bank_skip?` で自動スキップ:

| 条件 | スキップ理由 |
|------|------------|
| 金額が正（入金）| 収入・ATM入金・返金 |
| 入出金内容に「ラクテンカ」「楽天カ」を含む | クレカ引き落とし＝カードCSVと重複 |
| 入出金内容が「カード出金」または「カ−ド出金」で始まる | ATM現金引き出し（現金はチャットで記録） |
| 入出金内容に「預金利息」を含む | 利息収入 |
| 入出金内容に「給与」を含む | 給与収入 |

上記以外の出金はすべて取込む（JCBデビット・振込・税金等）。

### 4-4. Pasmoの扱い

楽天カードでチャージ → 楽天カードCSVに「Pasmoチャージ」として計上される。
Pasmo利用ごとの個別記録は不要（二重計上になる）。
**ただし**、SuicaやPasmo等でクレカチャージしない場合はAIチャットで記録。

### 4-5. 楽天証券の扱い

毎月5万円を楽天カードでクレカ積立 → 楽天カードCSVに自動計上（カテゴリ: 投資）。
証券口座の評価額は資産管理ページで月次管理。

## 6. AI機能

### 6-1. 自動カテゴリ分類
- CSV取込時、キーワードルールで分類できない店舗名をClaudeが一括分類
- `AiCategoryClassifierService` でバッチ処理

### 6-2. AIチャット（`/api/v1/ai/chat`）
- モデル: claude-opus-4-5
- Tool use: `record_transaction`（店舗名・金額・日付・支払方法・カテゴリIDを受け取りDBに保存）
- SSEでレスポンスを返す。tool_use時は `{ type: "tool_result", message: "✅ ..." }` を送信

### 6-3. 月次AIサマリ（`/api/v1/ai/monthly_summary`）
- AnalyticsPage「概要」タブのAIサマリカードで表示
- 当月の支出傾向・節約アドバイスを生成

## 8. DB 主要テーブル

| テーブル | 用途 | 備考 |
|---------|------|------|
| transactions | 取引明細 | upload_history_id で履歴と紐づく |
| categories | カテゴリマスタ（7件固定） | seeds.rb で初期投入 |
| category_rules | キーワード→カテゴリのルール | `priority` 昇順で適用。`after_save` でCSV自動更新 |
| upload_histories | CSVアップロード履歴 | `file_hash`（MD5）で重複インポートを検出。削除すると transactions も連鎖削除 |
| budgets | 月次予算 | month × category_id でユニーク |
| asset_accounts | 資産口座マスタ（6件固定） | `AssetAccount.seed!` で初期投入 |
| asset_snapshots | 月次資産残高スナップショット | asset_account_id × recorded_month でユニーク |

## 9. 月次の作業フロー

**支出記録**
1. 楽天カードCSVをダウンロード → アップロード
2. エポスカードCSVをダウンロード → アップロード
3. 楽天銀行CSVをダウンロード → アップロード（自動フィルタ適用）
4. 現金・Pasmo・PayPay払いをAIチャットで記録

**資産記録**
5. 月末に各口座の残高を資産管理ページに入力（楽天銀行・ゆうちょ・楽天証券・現金・PayPay・Pasmo）

**振り返り**
6. 詳細分析ページのAIサマリで当月を振り返る
