# 家計簿アプリ 要件定義書

**バージョン**: 1.0  
**作成日**: 2025年8月10日  
**更新日**: 2025年8月10日  

## 1. プロジェクト概要

楽天カードの利用明細 CSV ファイルを読み込み、カテゴリ別に自動分類して月毎の支出を積み上げ棒グラフで可視化する家計簿アプリケーション

### 1.1 背景・目的

楽天カード利用者が自身の支出状況を効率的に把握し、家計管理を改善することを目的とする。従来の CSV ファイルを Excel で開く作業を自動化し、直感的で視覚的な支出分析を提供する。

### 1.2 ユーザーペルソナ

- **主要ユーザー**: 楽天カードを利用している個人
- **特徴**: 家計管理に関心があり、定期的に支出を確認したい
- **スキルレベル**: 基本的な PC 操作ができる（CSV ファイルのダウンロード等）

### 1.3 主要ユースケース

1. **月毎支出の可視化**: 過去 1 年間の支出推移をグラフで把握
2. **カテゴリ別分析**: 食費、交通費等の内訳を確認
3. **詳細データ確認**: 特定の取引や期間の明細を検索・閲覧
4. **支出パターン分析**: 月平均、最大支出月等の統計情報を把握

## 2. 機能要件

### 2.1 主要機能

- 楽天カード CSV ファイルの読み込み
- 支出データの月毎集計
- 月毎支出額の棒グラフ表示

### 2.2 詳細機能

#### CSV ファイル読み込み機能

- ドラッグ&ドロップまたはファイル選択 UI
- CSV 形式・エンコーディングの検証
- BOM 付き UTF-8 の自動検出・処理
- データ解析・変換・エラーハンドリング

#### データ処理・分析機能

- 利用日による月毎グルーピング
- 支出金額の合計計算
- 店名による自動カテゴリ分類
- 統計データ生成（平均、最大・最小、取引回数）

#### グラフ表示機能

- **メイングラフ**: 積み上げ棒グラフ
  - 横軸：月（YYYY-MM 形式）
  - 縦軸：支出金額（円）
  - カテゴリ別色分け表示
  - インタラクティブな操作（ホバー、クリック）
- **期間選択機能**: 3ヶ月/6ヶ月/12ヶ月/全期間
- **凡例表示**: カテゴリ別の色と金額
- **カテゴリ表示制御**（オプション）: 
  - 個別カテゴリの表示・非表示切り替え
  - 凡例クリックによるカテゴリの一時非表示
  - 表示設定の保存

#### 詳細表示・編集機能

- **明細一覧表示**:
  - 取引明細一覧表示
  - カテゴリ別フィルタリング
  - 期間・金額・店名による検索
  - ページネーション（大量データ対応）

- **カテゴリ手動修正**:
  - 個別取引のカテゴリ変更（ドロップダウン選択）
  - 一括カテゴリ変更（複数選択での一括処理）
  - カテゴリ変更履歴の表示
  - 修正内容の保存・復元機能

- **分析表示**:
  - カテゴリ別円グラフ
  - 月次統計サマリー
  - 修正前後の比較表示

- **表示項目カスタマイズ機能**（オプション）:
  - 明細一覧の表示項目選択（利用日、店名、金額、カテゴリ等）
  - カラム表示・非表示の切り替え
  - カラム並び順の変更（ドラッグ&ドロップ）
  - ユーザー設定の保存・復元

#### カテゴリ自動分類・仕分け機能

- **自動分類エンジン**:
  - 店名・商品名のパターンマッチング
  - キーワード辞書による分類
  - 機械学習による分類精度向上（将来対応）
  
- **分類カテゴリ**:
  - 食費（スーパー、コンビニ、レストラン等）
  - 交通費（電車、バス、タクシー、ガソリン等）
  - 住宅費（家賃、住宅ローン、管理費、修繕費等）
  - 光熱・通信費（電気、ガス、水道、携帯、インターネット等）
  - 娯楽（映画、ゲーム、スポーツ等）
  - 投資・金融（証券、保険、銀行手数料）
  - ショッピング・サービス（衣類、家電、雑貨、美容、医療、教育等）
  - その他

- **手動修正機能**:
  - 自動分類結果の手動修正
  - ユーザー独自のカテゴリ作成
  - 学習機能（修正内容を次回分類に反映）
  
- **分類ルール管理**:
  - カスタムルールの作成・編集
  - 分類精度の確認・改善

## 3. 非機能要件

### 3.1 技術要件

#### フロントエンド
- **フレームワーク**: Vue.js 3 + JavaScript
- **グラフライブラリ**: Chart.js + vue-chartjs
- **ビルドツール**: Vite
- **CSS フレームワーク**: Tailwind CSS（予定）

#### バックエンド（必要に応じて）
- **API サーバー**: Ruby on Rails 7
- **高速処理が必要な場合**: Go（CSV 処理、データ集計）
- **データベース**: PostgreSQL または SQLite

#### CSV 処理
- **フロントエンド**: PapaParse（クライアントサイド処理）
- **バックエンド**: Ruby CSV ライブラリまたは Go の encoding/csv

### 3.2 ユーザビリティ要件

- 直感的なファイル選択 UI
- レスポンシブデザイン
- データ読み込み中の進捗表示

### 3.3 データ要件

- **対象データ量**: 20年分の取引明細（約24万件〜48万件を想定）
- **対応CSV形式**: 
  - **Phase 1**: 楽天カード CSV 形式
  - **将来対応予定**: 他社クレジットカード、銀行明細等（順次追加）
- **文字エンコーディング**: UTF-8 with BOM、Shift_JIS等の多様な形式に対応
- **データサイズ**: 50MB〜100MB の CSV ファイル処理
- **拡張性**: 新しいCSV形式の追加が容易な設計

### 3.4 パフォーマンス要件

- **CSV読み込み時間**: 20年分データを30秒以内で処理完了
- **グラフ描画時間**: データ表示まで5秒以内
- **画面遷移**: 1秒以内の応答時間
- **メモリ使用量**: ブラウザメモリ1GB以内での動作
- **大量データ対応**: 仮想化・ページネーションによる快適な操作

## 4. 制約事項

### 4.1 技術的制約

- **データ処理**: 大量データ（20年分）のため、必要に応じてバックエンド処理を採用
- **ブラウザ制限**: メモリ使用量とJavaScript処理性能の制限を考慮
- **CSV形式**: Phase 1では楽天カード形式のみ対応（将来的に他社対応予定）

### 4.2 セキュリティ・プライバシー制約

- **個人情報保護**: 金融データの適切な取り扱い
- **データ保存**: ユーザー同意なしにサーバーへデータ永続化しない
- **通信暗号化**: データ送信時はHTTPS必須

### 4.3 運用制約

- **オフライン動作**: インターネット接続不要での基本機能動作
- **ブラウザ対応**: モダンブラウザ（Chrome, Firefox, Safari, Edge）のみ
- **モバイル対応**: レスポンシブデザインによる基本的なモバイル表示

## 5. 実際の CSV データ形式

```
"利用日","利用店名・商品名","利用者","支払方法","利用金額","支払手数料","支払総額","支払月","8月支払金額","9月繰越残高","9月以降支払金額"
```

### CSV ファイルの特徴

- エンコーディング：UTF-8 with BOM（先頭に［文字）
- 区切り文字：カンマ（,）
- 引用符：ダブルクォート（"）で囲まれている
- 11 列の構造
- 利用金額は数値（カンマ区切りなし）
- 日付形式：YYYY/MM/DD

### 重要なカラム

- **利用日**：取引日（YYYY/MM/DD 形式）
- **利用店名・商品名**：店舗名や商品情報
- **利用金額**：支払金額（数値）
- **支払月**：8 月、9 月以降などの支払月情報
