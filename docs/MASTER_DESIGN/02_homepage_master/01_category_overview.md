## 1. カテゴリマスタの目的

### 1.1 トップページでの役割

カテゴリマスタは、**トップページ（HomePage）で支出データを分類・表示する基準**として機能します。

**主な用途**:
- 📊 **グラフ表示**: カテゴリ別の支出を円グラフ・棒グラフで表示
- 🎨 **色分け**: 各カテゴリに対応した色で視覚的に識別
- 📈 **集計**: カテゴリ別の合計金額・平均金額を計算
- 🔢 **表示順序**: `display_order`に基づいて一貫した順序で表示

### 1.2 設計方針

| 方針 | 説明 |
|------|------|
| **固定マスタ** | 初期値は固定で、ユーザーは追加・削除不可 |
| **視覚的識別** | 色・アイコンで直感的に識別可能 |
| **一貫性** | 全ページで同じカテゴリ定義を共有 |
| **拡張性** | 将来的にカスタマイズ機能を検討 |

## 3. 優先度・表示順序

### 3.1 display_order の役割

`display_order`は、**UI表示時の順序を制御**するためのフィールドです。

**特徴**:
- 数値が小さいほど先に表示される（昇順）
- トップページのグラフ凡例、カテゴリ一覧などで使用
- 一貫性を保つため、全ページで同じ順序を使用

### 3.2 表示順序の設定

```ruby
# display_order の値
1: 投資      # 最も優先度が高い
2: 食費
3: 日用品費
4: 娯楽費
5: 住宅費
6: 交通費
99: その他   # 最後に表示（予備カテゴリ）
```

**注意**: 「その他」は`display_order: 99`とすることで、常に最後に表示されます。

### 3.3 データ取得時の並び替え

```ruby
# Rails Model
scope :ordered, -> { order(:display_order, :name) }

# 使用例
categories = Category.ordered
# => [投資, 食費, 日用品費, 娯楽費, 住宅費, 交通費, その他]
```

## 4. UI表示制御

### 4.1 トップページでの使用箇所

#### 4.1.1 グラフ表示（ExpenseChart.vue）

**用途**: カテゴリ別支出を円グラフ・棒グラフで表示

**実装例**:
```javascript
// カテゴリの色をグラフに適用
const chartData = {
  labels: categories.map(c => c.name),
  datasets: [{
    data: categoryAmounts,
    backgroundColor: categories.map(c => c.color) // 色コードを使用
  }]
}
```

**表示順序**: `display_order`順に表示

#### 4.1.2 統計カード（SummaryCards.vue）

**用途**: カテゴリ別の統計情報を表示

**実装例**:
```javascript
// カテゴリごとの合計金額を計算
const categorySummary = categories.map(category => ({
  name: category.name,
  color: category.color,
  icon: category.icon,
  total: transactions
    .filter(t => t.category_id === category.id)
    .reduce((sum, t) => sum + t.amount, 0)
}))
```

#### 4.1.3 カテゴリ別集計

**用途**: APIから取得したデータをカテゴリ別に集計

**APIエンドポイント**:
```
GET /api/v1/transactions/summary
```

**レスポンス例**:
```json
{
  "summary": {
    "by_category": [
      {
        "category_id": 2,
        "category_name": "食費",
        "category_color": "#4BC0C0",
        "total_amount": 50000,
        "transaction_count": 45
      },
      ...
    ]
  }
}
```

### 4.2 色・アイコンの使用

#### 色コード形式

- **形式**: `#RRGGBB` (16進数)
- **例**: `#4BC0C0` (シアン)
- **用途**: Chart.jsのグラフ、CSS背景色、ボーダー色

#### アイコン

- **形式**: 絵文字（Unicode）
- **例**: 🍽️, 🚗, 🏠
- **用途**: UI表示、カテゴリ識別の補助

### 4.3 レスポンシブ対応

トップページのグラフ・統計カードは、カテゴリマスタの情報を基に動的に生成されます。

**考慮事項**:
- モバイル表示時も`display_order`順を維持
- 色のコントラストを確保（アクセシビリティ）
- アイコンが表示されない環境でも識別可能

## 5. データフロー

### 5.1 トップページ表示時の流れ

```
1. ページロード
   ↓
2. API: GET /api/v1/categories
   → カテゴリ一覧取得（display_order順）
   ↓
3. API: GET /api/v1/transactions/summary
   → カテゴリ別集計データ取得
   ↓
4. フロントエンド: データ結合
   → カテゴリ情報 + 集計データ
   ↓
5. UI表示
   → グラフ（色・順序）
   → 統計カード（色・アイコン）
```

### 5.2 カテゴリマスタの更新タイミング

**現在**: カテゴリマスタは固定のため、更新は不要

**将来的**: カテゴリカスタマイズ機能が追加された場合
- ユーザーがカテゴリを追加・編集した際に、トップページの表示も自動更新
