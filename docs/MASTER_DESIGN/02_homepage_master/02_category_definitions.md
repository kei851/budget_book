## 1. 概要

このドキュメントでは、トップページで使用される7つのカテゴリについて、それぞれの詳細な定義・該当支出例・キーワード例を説明します。

**対象読者**:
- 開発者: カテゴリ分類ロジックの実装
- ユーザー: カテゴリの理解とルール追加の参考

## 2. 各カテゴリの詳細説明

### 2.1 💰 投資 (Investment)

#### 基本情報

- **ID**: 1
- **表示順序**: 1
- **色コード**: `#FF6384` (ピンク/赤)
- **アイコン**: 💰
- **説明**: 証券、保険、銀行手数料

#### 該当する支出例

**証券関連**:
- 楽天証券での取引手数料
- SBI証券での取引手数料
- マネックス証券での取引手数料
- 投資信託の購入・売却手数料
- 株式の購入・売却手数料

**保険関連**:
- 生命保険料
- 損害保険料（自動車保険、火災保険等）
- 医療保険料
- 学資保険料

**金融関連**:
- 銀行振込手数料
- クレジットカード年会費
- 投資信託の信託報酬
- 仮想通貨取引手数料

#### 対象キーワード例

分類ルールマスタに登録されるキーワード例:
- `楽天証券`, `SBI証券`, `マネックス証券`
- `保険`, `生命保険`, `損害保険`
- `銀行手数料`, `振込手数料`
- `クレジットカード`, `年会費`

#### 除外すべき支出

- **給与**: 収入なので支出カテゴリに含めない
- **配当金**: 収入なので別管理
- **投資商品の購入額**: 資産形成のため、別カテゴリで管理することを検討

### 2.7 ❓ その他 (Other)

#### 基本情報

- **ID**: 7
- **表示順序**: 99（最後に表示）
- **色コード**: `#C9CBCF` (グレー)
- **アイコン**: ❓
- **説明**: 上記に該当しないもの

#### 該当する支出例

**分類不明な支出**:
- 自動分類が失敗した取引
- どのカテゴリにも該当しない支出

**複合カテゴリ**:
- 複数のカテゴリにまたがる支出
- 分類が難しい支出

**初期状態で未分類**:
- 新規に追加された取引で、まだ分類ルールが登録されていない店舗

#### 対象キーワード例

**注**: このカテゴリには通常、キーワードルールは登録しません。自動分類が失敗した場合の予備カテゴリとして機能します。

#### 補足

- **分類精度向上**: ユーザーが「その他」に分類された取引を確認し、適切なカテゴリに手動で分類することで、分類精度を向上させることができます
- **ルール追加のヒント**: 「その他」に分類された取引が多い店舗があれば、新しい分類ルールを追加する候補となります

## 3. グラフ表示時の色分け

### 3.1 色コード一覧

| カテゴリ | 色コード | RGB値 | 視認性 |
|---------|---------|-------|--------|
| 投資 | `#FF6384` | RGB(255, 99, 132) | 高 |
| 食費 | `#4BC0C0` | RGB(75, 192, 192) | 高 |
| 日用品費 | `#9966FF` | RGB(153, 102, 255) | 高 |
| 娯楽費 | `#36A2EB` | RGB(54, 162, 235) | 高 |
| 住宅費 | `#FF9F40` | RGB(255, 159, 64) | 高 |
| 交通費 | `#FFCE56` | RGB(255, 206, 86) | 高 |
| その他 | `#C9CBCF` | RGB(201, 203, 207) | 中 |

### 3.2 Chart.jsでの使用

トップページのグラフ（ExpenseChart.vue）では、以下のように色コードを使用します：

```javascript
const chartOptions = {
  backgroundColor: [
    '#FF6384', // 投資
    '#4BC0C0', // 食費
    '#9966FF', // 日用品費
    '#36A2EB', // 娯楽費
    '#FF9F40', // 住宅費
    '#FFCE56', // 交通費
    '#C9CBCF'  // その他
  ]
}
```

### 3.3 色の選定理由

- **視認性**: グラフ上で識別しやすい色を選択
- **コントラスト**: 背景色（白）とのコントラストを確保
- **色覚対応**: 色覚多様性を考慮した色の組み合わせ

## 4. まとめ

### 4.1 カテゴリ定義の重要性

各カテゴリの定義を明確にすることで：

- **分類精度の向上**: ユーザーが適切なルールを追加できる
- **一貫性の確保**: 開発者・ユーザー間でカテゴリの理解が一致
- **拡張性**: 将来的にカテゴリを追加する際の基準となる
