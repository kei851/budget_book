## 1. 概要

### 1.1 初期ルールセットの目的

**初期ルールセット**は、システム導入時に提供される標準的な分類ルールです。

**主な目的**:
- 🚀 **すぐに使える**: システム導入直後から自動分類が可能
- 📊 **一般的な店舗をカバー**: 楽天カード利用者の一般的な店舗をカバー
- 🎯 **分類精度のベースライン**: ユーザーが追加・編集する際の基準

### 1.2 ルール数

| カテゴリ | 初期ルール数 | 優先度範囲 |
|---------|------------|-----------|
| 投資 | 約 5 キーワード | 100 ～ 110 |
| 食費 | 約 40 キーワード | 100 ～ 140 |
| 日用品費 | 約 20 キーワード | 100 ～ 120 |
| 娯楽費 | 約 20 キーワード | 100 ～ 120 |
| 住宅費 | 約 5 キーワード | 100 ～ 110 |
| 交通費 | 約 20 キーワード | 100 ～ 120 |

## 3. ルール選定基準

### 3.1 選定基準

| 基準 | 説明 | 例 |
|------|------|-----|
| **楽天カード利用者の一般的な店舗** | 楽天カード利用者が頻繁に利用する店舗 | セブンイレブン、ローソン |
| **店舗名の特徴的な部分** | 店舗名に含まれる特徴的な文字列 | `セブン`、`JR` |
| **一般的な略称** | 店舗名の一般的な略称 | `マック`、`スタバ` |
| **ブランド名** | ブランド名やチェーン名 | `イオン`、`無印良品` |

### 3.2 優先度設定基準

| 優先度範囲 | 基準 | 例 |
|-----------|------|-----|
| **130 以上** | 特定の主要チェーン店 | セブンイレブン、マクドナルド |
| **120 ～ 129** | 特定の店舗・ブランド | ローソン、イオン |
| **110 ～ 119** | 一般的な店舗タイプ | ガソリン、駅 |
| **100 ～ 109** | 汎用的なキーワード | スーパー、コンビニ |

### 3.3 キーワード重複排除

- **同じキーワードの重複**: UNIQUE 制約により防止
- **類似キーワード**: より具体的なキーワードを優先（例: `セブンイレブン` と `セブン`）

## 4. 初期化スクリプト

### 4.1 Rails Seed スクリプト例

```ruby
# db/seeds/category_rules.rb

category_rules_data = [
  # 投資
  { keyword: '楽天証券', category_name: '投資', priority: 110 },
  { keyword: 'SBI証券', category_name: '投資', priority: 110 },
  { keyword: 'マネックス証券', category_name: '投資', priority: 110 },
  { keyword: '保険', category_name: '投資', priority: 105 },
  { keyword: 'クレジットカード', category_name: '投資', priority: 104 },
  
  # 食費
  { keyword: 'マイバスケット', category_name: '食費', priority: 140 },
  { keyword: 'セブンイレブン', category_name: '食費', priority: 135 },
  { keyword: 'ローソン', category_name: '食費', priority: 135 },
  { keyword: 'ファミマ', category_name: '食費', priority: 135 },
  { keyword: 'マクドナルド', category_name: '食費', priority: 130 },
  { keyword: 'スターバックス', category_name: '食費', priority: 130 },
  # ... その他のキーワード
]

puts "分類ルールデータを投入中..."

category_rules_data.each do |rule_data|
  category = Category.find_by(name: rule_data[:category_name])
  next unless category
  
  CategoryRule.find_or_create_by!(
    keyword: rule_data[:keyword],
    category: category
  ) do |rule|
    rule.priority = rule_data[:priority]
  end
  
  print "."
end

puts "\n#{CategoryRule.count} 個の分類ルールが作成されました。"
```

### 4.2 実行方法

```bash
# Seed スクリプト実行
rails db:seed

# または、カテゴリルールのみ投入
rails runner "load 'db/seeds/category_rules.rb'"
```
